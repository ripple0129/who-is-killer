# Who Is Killer — 遊戲設計文件

## 概要

一款線上多人 AI 推理遊戲。3 位人類偵探各帶 1 名 AI 嫌疑犯進場，其中 1 名 AI 是兇手。偵探透過質問、推理、討論，最終投票逮捕兇手。

## 基本設定

- **玩家數：** 3 人（各帶 1 AI，共 6 角色）
- **遊戲時長：** 約 20 分鐘
- **勝利條件：** 偵探投票淘汰兇手 AI → 偵探勝；兇手 AI 存活 → 兇手方勝

## 角色

| 角色 | 數量 | 說明 |
|------|------|------|
| 偵探（人類） | 3 | 負責質問、推理、投票 |
| 無辜嫌疑犯（AI） | 2 | 有自己的秘密但不是兇手 |
| 兇手嫌疑犯（AI） | 1 | 需要說謊隱瞞，不被抓到 |

- 每位偵探負責一名嫌疑犯
- 偵探不知道自己的嫌疑犯是否為兇手

## 遊戲空間

### 公開審訊室（主畫面）
- 所有人都在場
- AI 嫌疑犯在此發言、被質問
- 線索在此公布

### 私人偵訊室（1 對 1）
- 偵探與自己負責的嫌疑犯
- 其他人看不到對話

### 偵探密室
- 僅 3 位人類偵探
- AI 嫌疑犯看不到
- 交換情報、討論策略

## 遊戲流程

### Phase 0 — 開場（1 分鐘）
- 系統展示案件背景：死者、地點、死因
- 所有人看到相同資訊

### Phase 1-3 — 調查回合（每回合約 5 分鐘，共 3 回合）

每回合流程：
1. **線索公布** — 系統揭露一條新線索（鑑識報告、監視器、通訊紀錄等）
2. **AI 公開供詞** — 3 名嫌疑犯各說一段話
3. **公開質問** — 偵探可對任意嫌疑犯提問，AI 即時回答
4. **私人偵訊** — 偵探與自己的嫌疑犯 1 對 1 對話
5. **偵探密室** — 偵探之間討論（AI 不在場）

### Phase 4 — 最終辯論（1 分鐘）
- 3 名嫌疑犯做最後陳述

### Phase 5 — 偵探最終會議（2 分鐘）
- 偵探密室最後討論

### Phase 6 — 投票（1 分鐘）
- 每位偵探投一票，選出認為是兇手的嫌疑犯
- 票數最多者被逮捕
- **平票處理：** 加一輪快速辯論再投

### Phase 7 — 真相揭曉
- 公布兇手身分、完整故事、動機

## 計分

| 結果 | 得分 |
|------|------|
| 投中兇手 | +3 分 |
| 沒投中但兇手被淘汰 | +1 分 |
| 兇手存活 | 兇手方偵探 +2 分，其他人 0 分 |

## 劇本生成

每場遊戲由 AI（LLM）動態生成劇本，包含：

- **案件背景** — 時間、地點、死者、死因
- **3 名嫌疑犯設定** — 名字、身分、與死者關係、真實行蹤、隱瞞的事
- **兇手指定** — 哪位是兇手、動機、手法
- **線索池** — 3 條逐回合揭露的線索
- **AI System Prompts** — 每名嫌疑犯的角色指令
- **真相揭曉文本** — 結局敘事

### 場景多樣性

透過隨機組合維度產生不重複劇本：
- 地點：豪宅、郵輪、醫院、學校、辦公大樓、夜店、露營地...
- 時代：現代、1920s、古代、近未來
- 死因：毒殺、刺殺、墜樓、溺水、勒斃...
- 關係：家族、同事、同學、鄰居、陌生人

## 技術架構

- **Monorepo** 結構
- **前端：** Next.js + TypeScript + Zustand + Tailwind CSS
- **後端：** Fastify + TypeScript + Drizzle ORM + PostgreSQL
- **即時通訊：** WebSocket
- **AI：** 可抽換 LLM provider（OpenAI / Claude / 其他）
- **部署：** 獨立專案，之後串接 Arinova SDK 導入玩家與 AI
